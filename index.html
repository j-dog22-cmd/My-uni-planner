Step-by-Step Guides
01
Security & Forensic Tools — Case Investigation
50% of module Due: 2 Apr ~15–20 hrs
▶
Submit: ① Provenance Report (40% of marks)   ② Contemporaneous Notes (60% of marks)
1
Download everything from Blackboard
Forensic image (.E01), case scenario document, and the Contemporaneous Notes template from the assessment section. Use the template — it's mandatory.
2
Open the template before touching the evidence
Start filling in your notes from the very first action. Document the acquisition. Record both the acquisition hash and verification hash immediately — failing this is a common mark-loser.
3
Systematic analysis — apply each lab topic to the image
File system → Browser history → Documents → Registry → Encrypted volumes/passwords → Steganography. Work through each area as you learn it in labs.
4
Bookmark evidential artefacts in EnCase
Only bookmark items relevant to the case. Each must have all 9 required fields: Name, Is Deleted, File Created, Last Written, Last Accessed, Logical Size, Physical Sector, Full Path, MD5 Hash.
5
Generate Provenance Report using EnCase's report template
Auto-generated from your bookmarks — do NOT manually type provenance blocks. Autopsy reports also accepted.
6
Write up Contemporaneous Notes thoroughly
Every action documented. Every non-action justified. Independent examiner must be able to repeat your exact examination. This is 60% of your marks.
7
Submit both documents (Word or PDF) before 2 Apr 14:00
Keep backups in at least 2 places — the brief explicitly warns you about this. 48-hour late window available if needed.
💡 You can score well without finding every piece of evidence. The contemporaneous notes (60% of marks) reward thorough process documentation — even documenting a negative result ("I searched X and found nothing relevant") shows good practice and earns credit.

100% of module Due: 23 Apr ~12–15 hrs
▶
Submit: ① Video link (max 15 min)   ② Ducky Script .docx   ③ PowerPoint slides
1
Pick up Raspberry Pi Pico from a lab session

2

3
Design a realistic attack scenario
Credential harvesting, persistence, data exfiltration, privilege escalation. Avoid pranks. Plan a full attack chain. Think: what TTPs does this cover?
4
Test simple scripts first, then build up
Confirm the Pico works with a basic script before building the full attack. Always test in a VM.
5
Write and test the full Ducky Script + supporting scripts
Comment every section. Test end-to-end. Do NOT use automated tools to generate it — prohibited and specifically assessed for originality.
6
Map attack to MITRE ATT&CK (15% of marks)
Specific TTP IDs for each stage. Also map your mitigations. Be precise — vague mappings lose marks.
7
Build PowerPoint slides
Scenario + target, attack design, technical setup, MITRE mapping diagram, stage-specific mitigations, CIA triad impact. Use diagrams — presentation is 10% of marks.
8
Practice and time your narration
Max 15 minutes — going over means it won't be marked beyond that. Structure: intro (1 min) → attack design (3 min) → live demo (5 min) → MITRE (3 min) → mitigations (3 min).
9
Record the final video and upload
Panopto or OBS. Show your face. Screen capture the live attack. Last edit must be before the submission deadline.
10
Submit all 3 items before 23 Apr 14:00
Test video link from another device first. ① Video link ② Ducky Script .docx ③ PowerPoint. Confirm Blackboard receipt.
💡 The mitigations section (15% of marks) is commonly done badly. Generic advice like "update your software" won't score well. Each mitigation must directly address a specific stage in YOUR attack chain and be mapped to MITRE where possible.
03
Data Science Task 4 — Security Dashboard
20% of module Due: 7 May ~8–10 hrs
▶
Submit: ① Data Specification Document   ② Python dashboard prototype
1
Read the full task spec and understand the marking criteria
4 monitoring areas. 20 marks total. Know what earns what before starting.
2
Write the Data Specification Document (4 marks)
For each of the 4 categories (Network, Endpoints, Backups, Staff Readiness), identify 3+ metrics. Each row needs: subcategory, metric/format, frequency, collection mode, mechanism, recorded at, storage.
3
Create 2+ dummy CSV files
e.g. network_traffic.csv and endpoint_data.csv with 5–10 rows of realistic-looking dummy values. No real data needed.
4
Plan your 6 charts on paper first
Decide layout before coding. You need ≥1 geo-spatial (map) and ≥1 interactive. Sketch placement — saves time when building.
5
Build the dashboard chart by chart
Use Streamlit + Plotly. Build and test each chart individually before combining. Don't try to do everything at once.
6
Add geo-spatial visualisation (2 marks)
Use Plotly's px.scatter_geo or px.choropleth. Map something meaningful like network traffic source countries.
7
Add interactive visualisation (2 marks)
Streamlit: st.selectbox or st.slider to filter chart data. Dash: @app.callback with dropdown input.
8
Polish the design (4 marks)
Consistent colour scheme, readable fonts, clear titles on every chart, important info placed prominently. Worth as much as the entire data spec document — don't skip it.
9
Final marking criteria check before submitting
✓ 3 metrics × 4 categories ✓ 6 charts ✓ 2+ input files ✓ 1 geo-spatial ✓ 1 interactive ✓ Effective design and layout
💡 Streamlit is much faster to set up than Dash if you're new to dashboarding — working prototype in under an hour. Plotly Express handles both geo-spatial and interactive charts with very minimal code.
